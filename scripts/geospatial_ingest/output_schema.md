# Geospatial Manifest Output Schema

**Schema ID:** `sais.geospatial_manifest.v1`

This document defines the schema for the `manifest.json` file generated by the Geospatial Ingest Pipeline. This manifest serves as the formal bridge between the offline processing step and the Farm Digital Twin running on the SAIS Node.

## Example Payload

```json
{
  "schema": "sais.geospatial_manifest.v1",
  "farm_id": "local",
  "generated_at": "2026-05-01T20:00:00Z",
  "input_layers": {
    "dem": "dem.tif",
    "soils": "soils.geojson"
  },
  "derived_layers": {
    "slope": "slope.tif",
    "aspect": "aspect.tif",
    "flow_accumulation": "flow_accumulation.tif",
    "soils_hydrologic_group": "soils_hydrologic_group.tif",
    "runoff_risk": "runoff_risk.tif"
  },
  "assumptions": [
    "runoff_risk is a heuristic susceptibility index",
    "combined hydrologic groups A/D, B/D, C/D treated conservatively as D",
    "risk does not yet include vegetation cover or rainfall intensity"
  ]
}
```

## Fields

- **`schema`** (string): The schema version identifier. Must be `sais.geospatial_manifest.v1`.
- **`farm_id`** (string): The identifier of the farm or domain. Defaults to `local`.
- **`generated_at`** (string): ISO-8601 UTC timestamp of pipeline execution.
- **`input_layers`** (object): Map of expected input source files and their basenames.
- **`derived_layers`** (object): Map of generated GeoTIFF files. The SAIS dashboard uses these keys to mount the static overlays.
  - `slope`: Degrees (0-90)
  - `aspect`: Degrees (0-360)
  - `flow_accumulation`: Number of upstream contributing pixels.
  - `soils_hydrologic_group`: Integer mapping of USDA classes (1=A, 2=B, 3=C, 4=D)
  - `runoff_risk`: 0-100 heuristic index fusing slope, soil impermeability, and flow accumulation.
- **`assumptions`** (array of strings): Human-readable caveats detailing the methodological limitations of the derived layers. Crucial for ensuring the intelligence layer treats these as proxies rather than absolute physical models.
