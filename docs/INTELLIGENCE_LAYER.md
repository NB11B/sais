# SAIS Intelligence Layer

**Version:** 0.1.0-draft
**Status:** Active Development

---

## Overview

The Sovereign Ag-Infrastructure Stack (SAIS) generates a massive volume of high-fidelity data at the edge: geometric phase shifts from the NGC kernel, spectral resonance data from the Adaptive Engine, and traditional SCADA telemetry (moisture, temperature, flow rates). 

If this raw data is simply passed to the farmer, it creates cognitive overload. The **Intelligence Layer** is the on-device decision engine that ingests these disparate sensor streams, synthesizes them into actionable intelligence, automates routine control decisions, and surfaces natural-language alerts to the farmer via the C2 Dashboard.

Crucially, this intelligence runs **entirely at the edge**. It does not rely on cloud APIs, ensuring the farm remains operational and sovereign even when internet connectivity is severed.

---

## 1. Architecture of the Intelligence Layer

The Intelligence Layer sits on the SCADA/Compute board (e.g., the UNO Q's Cortex-A53 or the i.MX 8M) and operates as a unified containerized service. It consists of three primary components:

### 1.1 The Edge Inference Engine (TinyML)
Raw telemetry and spectral data are fed into lightweight, pre-trained machine learning models running via TensorFlow Lite for Microcontrollers (TFLM) or NXP eIQ [1]. 
- **Anomaly Detection:** Autoencoders continuously monitor the baseline "healthy" state of the farm (e.g., normal pump vibration, expected soil moisture decay curves). Any deviation from this baseline triggers an anomaly flag [2].
- **Predictive Maintenance:** By analyzing the FFT output from the Adaptive Engine, the inference engine predicts bearing wear or structural fatigue before a mechanical failure occurs.

### 1.2 The Local Knowledge Graph (RAG)
The node maintains a local vector database containing agronomic best practices, crop-specific stress indicators, the farmer's historical logs, and equipment manuals. 
- When an anomaly is detected (e.g., a sudden drop in soil moisture combined with a specific NDVI spectral shift), the system queries this local knowledge base to contextualize the anomaly [3].

### 1.3 The On-Device LLM (Advisory Agent)
A highly quantized Large Language Model (e.g., Llama-3-8B-Instruct quantized to 4-bit, or a smaller 2B parameter model) runs locally on the compute board's NPU/GPU [4]. 
- It ingests the anomaly flag and the retrieved context from the RAG system.
- It generates a concise, natural-language alert for the farmer.

## 2. The Decision Loop: From Sensor to Action

The Intelligence Layer operates on a continuous OODA loop (Observe, Orient, Decide, Act):

1. **Observe:** The Controller Layer (ESP32/STM32) streams raw data (moisture = 18%, pump vibration = 120Hz peak, NGC phase = 0.8) to the Compute Layer via DDS.
2. **Orient:** The Edge Inference Engine flags the 120Hz vibration as an anomaly (bearing wear). The RAG system retrieves the pump's maintenance manual and historical failure rates.
3. **Decide:** The On-Device LLM synthesizes this: *"Irrigation Pump 3 is showing early signs of bearing failure. Estimated time to critical failure: 48 hours. Soil moisture in Sector A is adequate for the next 72 hours."*
4. **Act:** 
   - **Automated:** The system automatically throttles Pump 3 to 70% capacity to extend its lifespan and routes Sector A irrigation to a backup pump.
   - **Advisory:** The LLM pushes the natural-language alert to the farmer's C2 Dashboard, recommending a bearing replacement during the next maintenance window.

---

## 3. The Farmer's Interface (C2 Dashboard)

The farmer interacts with the Intelligence Layer through the C2 Dashboard, which shifts from a traditional SCADA interface (dials and graphs) to an **Advisory Interface**.

- **The Daily Briefing:** Upon logging in, the farmer receives a synthesized summary of the farm's health, generated by the local LLM. *(e.g., "Good morning. All systems nominal. Sector B irrigation completed successfully. Note: Spectral analysis indicates early nitrogen stress in the north quadrant of Sector C.")*
- **Conversational Querying:** The farmer can ask the system questions in natural language. *(e.g., "Why did you throttle Pump 3 last night?")* The LLM responds using the local logs and RAG context, explaining its automated decisions transparently.

---

## 4. Hardware Requirements

To support the Intelligence Layer, the SCADA/Compute board must have sufficient resources for edge AI:

- **Compute:** Quad-core ARM Cortex-A53 or better (e.g., UNO Q QRB2210, Raspberry Pi CM4, NXP i.MX 8M).
- **Memory:** Minimum 2GB RAM (4GB recommended for running the local LLM and vector database concurrently).
- **Acceleration:** An integrated NPU (Neural Processing Unit) or GPU (like the Adreno on the UNO Q) is highly recommended to run the LLM and TFLM models efficiently without thermal throttling.
