# Farm Digital Twin Core Graph

This package (`farm_twin`) serves as the core spatial and relational intelligence layer for SAIS. It is an in-memory or SQLite-backed graph database that bridges static geospatial intelligence with real-time sensor telemetry.

## Concept

Geospatial data alone is static. Sensor data alone lacks spatial context. The Farm Graph connects them:
- **Farm** contains **Fields**
- **Fields** contain **Management Zones**
- **Zones** have spatial characteristics (e.g., Runoff Risk, Soil Type)
- **Sensors** are deployed in **Zones**
- Sensors produce **Observations**

By querying this graph, we can generate contextual intelligence cards. For instance, if a sensor reports falling soil moisture, the graph knows if that sensor is in a zone with a high risk of runoff, and issues a targeted advisory.

## Components

- `storage.py`: SQLite backend mapping nodes, edges, observations, and cards.
- `models.py`: Dataclass schema definition for all farm objects.
- `graph.py`: Central manager for adding and querying objects.
- `ingest_geospatial.py`: Loads the manifest generated by the offline pipeline.
- `ingest_observation.py`: Loads the farm hierarchy and live telemetry.
- `queries.py` and `cards.py`: Logic engines that analyze the graph to produce operator advisories.
